ubuntu16循环登录问题

发表于 2019-07-05 更新于 2019-07-25

之前通过网上各种方法，找到了问题所在，以及解决问题的方法，可惜在官网只能找到tesla k80相关的deb包，然而惊喜来的太突然，发现只要操作系统选择Linux 64位（之前一直选择Linux ubuntu16.4），就可下载到run文件（莫名其妙），这样安装驱动就可去除OpenGL选项，从而解决问题！

方法转载自https://blog.csdn.net/qq_38451119/article/details/81008165

###问题：

装了nvidia显卡驱动后开机一直处于循环登录界面，密码输入正确也是进不去，然后卸载nvidia显卡驱动。

###原因：NVIDIA 驱动所致

这时候是没办法进入图形界面的，只能苦逼的用命令行模式来解决。

###解决方法：

第一步：进入命令行模式——关闭显卡驱动——卸载Nvidia驱动
（1）同时按下 CTRL+ALT+F1 进入命令行模式，

（2）关闭显卡驱动，输入：

$sudo service lightdm stop
（这是因为我们之后需要在ubuntu上重装Nvidia的显卡驱动，需要关闭图形界面才能安装驱动）

（3）卸载NVIDIA驱动，输入：

$sudo apt-get purge nvidia*
这条命令会卸载与NVIDIA相关的一切，注意如果此时重新启动电脑可能会导致无法进入系统。

第二步：安装Nvidia驱动

第三步：

命令行模式输入：
$dpkg --reconfigure lightdm lightdm-gtk-greeter

第四步：重启

命令行模式输入：
$reboot
之后系统会重启，这时我们就发现用户登录界面死循环问题得到解决啦！！

搭建ubuntu服务器

发表于 2019-06-25 更新于 2019-07-25

Nvidia driver 410 + CUDA10.0 + cudnn7.4.2 + tensorflow-gpu1.13.1 + python3.6.8 on Ubuntu16.04(by ssh)

###前言：

由于本人近期准备在深度学习上搞搞事情，正好实验室有一台搭载4个k80显卡的服务器，于是就开始了入坑的旅程。
刚开始接手这个服务器，师兄给告知不知为什么在ubuntu登陆界面会出现循环登陆的问题，由于linux小菜的我并不知道师兄们做了啥，于是在师兄备份过自己实验数据的情况下重装系统，由于对UI的极度热爱，又因为了解到ubuntu的桌面版和服务器版功能相差无几，而且自带界面，因此我选择给服务器装了ubuntu16.04-desktop版。

安装完系统，我们进入深度学习环境搭建的正题。
###摘要
先看一下cuda, cudnn, tensorflow, python等对应版本（可以说这是环境搭建最烦的地方，一点都不智能～）

  • 1.13.1及以后版本已经可以在python3.7上运行，无需再手动设置python3.6版本或降级。当然我们还是采用python3.6比较稳定，使用anaconda进行管理多版本python环境也很方便。
  • Tensorflow 【1.13.1】 只能兼容 【CUDA 10.0】！注意是10.0，不是10.1，更不是官网说的9.0！
  • GPU驱动需要在 【410.xx】 及以上，注意匹配 【Notebook/Desktop】 版本。
  • CUDA 9.0和10.0 可以共存，但是注意设置环境变量，调用正确的版本。

####本服务器硬件

  • cpu: Intel(R) Xeon(R) CPU E5-2640 v4 @ 2.40GHz
  • gpu: 4 * Tesla k80 + 联想服务器自带的垃圾集成显卡(compatible controller: Matrox Electronics Systems Ltd. G200eR2 (rev 01))
  • RAM: 64GB
  • 安装平台：ubuntu16.04 + Anaconda3-2019.03-Linux-x86_64.sh

###anaconda安装
在官网上下载到Anaconda3-2019.03-Linux-x86_64.sh后，cd到下载目录：
$sudo bash Anaconda3-2019.03-Linux-x86_64.sh
如果提示输入安装路径，可以按默认路径，也可以自行设置。
一个使用Linux的良好习惯，把安装的所有用户组共享的应用都放在 /usr/local 下。所以我设置的安装路径是：
/usr/local/anaconda3
这里我们选择默认，安装在当前目录下。
一直按Enter直到提示输入”yes“ or “no”，然后输入“yes”。
如果提示是否Configure，选择“yes”。如果提示是否安装VS之类的，根据需要决定，我选择“no”。
####配置python环境变量

测试：

  1. 输入python查看版本
  2. quit()或exit()退出python

###安装nvidia驱动（前方高能）
由于本级刚开始使用的是ubuntu自带的通用驱动（很垃圾，整个图像化界面十分卡顿），没有安装nvidia驱动，nvidia-smi命令无法使用。于是我开始了无止境的驱动安装卸载。。。
为什么呢？因为我按照网上教程成功在nvidia官网下载对应k80的410驱动，但是！我以为这个同笔记本电脑一样同时支持显示，于是按照要求重启，于是就出现了登陆界面循环登入，也终于明白师兄们的情况怎么来的。接着使用网上一系列方法：关闭lightdm，禁用Nouveau驱动（自带的）等等。问题却无动于衷。于是只有进入ttys（即终端）卸载了nvidia驱动，切换回了自带的驱动。终于又回到了熟悉的UI，又在网上搜罗着各种方法，尝试着找到根源，并解决这个问题。于是在服务器前待了一天又一天，终于在第二天晚上实在受不了那10000+分贝的声响，我放弃了解决这个问题。
但也算找到了原因所在：k80 这种服务级显卡对界面本来就支持不好，所以换显卡驱动解决不了这种驱动（也就是之前能显示是用的服务器自带的集成显卡）。

FPN for OD(CVPR2017)阅读笔记

发表于 2019-05-06

原文地址：Feature pyramid networks for object detection

代码地址：fpn.pytorch

简介

特征金字塔适用于不同尺度的对象的检测。但最近深度学习对象检测器避免了金字塔表示，部分原因是其密集型的计算与内存。原文利用深度卷积网络的固有多尺寸金字塔层次来构造具有边际额外成本的特征金字塔，即开发了一种具有横向连接的自上而下的架构，用于在所有尺度上构建高级语义特征图。这种被称为特征金字塔网络（FPN）的体系结构在许多应用程序中作为通用特征提取器，结果体现出显著的提升。在COCO2016挑战赛时，作者在基本的Faster R-CNN中使用了FPN结构，在检测精度上超过了任意一个单模型的结果，并刷新了单模型的记录。

算法概述

简要说明：

  • (a)用图像金字塔生成特征金字塔
  • (b)只在特征最上层预测
  • (c)特征层分层预测
  • (d)FPN从高层携带信息传给底层，再分层预测

不难看出，作者的创新之处就在于既使用了特征金字塔（利用CNN的金字塔层次结构特性），又搞了分层预测。

背景

多尺度目标检测是计算机视觉领域的一个基础且具挑战性的课题。在图像金字塔基础上构建的特征金字塔（featurized image pyramids ,Figure1[a]）是传统解决思路，具有一定意义的尺度不变性。直观上看，这种特性使得模型可以检测大范围尺度的图像。

Featurized image pyramids 主要在人工特征中使用，比如DPM就要用到它产生密集尺度的样本以提升检测水平。目前人工特征式微，深度学习的CNN特征成为主流，CNN特征的鲁棒性很好，刻画能力强。即使如此，仍需要金字塔结构去进一步提升准确性，尤其在多尺度检测上。金字塔结构的优势是其产生的特征每一层都是语义信息加强的，包括高分辨率的低层。

对图像金字塔每一层都处理有很大的局限性，首先运算耗时会增加4倍，训练深度网络的时候太吃显存，几乎没法用，即使用了，也只能在检测的时候。因为这些原因，Fast/Faster R-CNN 都没使用featurized image pyramids。

当然，图像金字塔并不是多尺度特征表征的唯一方式，CNN计算的时候本身就存在多级特征图（feature map hierarchy），且不同层的特征图尺度就不同，形似金字塔结构（Figure1[b]）。结构上虽不错，但是前后层之间由于不同深度（depths）影响，语义信息差距太大，主要是高分辨率的低层特征很难有代表性的检测能力。

SSD(The Single Shot Detector)方法便是利用CNN的多级特征图，为了避免利用太低层的特征，SSD从VGG网络偏后的conv4_3开始，又往后加了几层，分别抽取每层特征，进行综合利用（Figure1[c]）。但是SSD对于高分辨率的底层特征没有再利用，而这些层对于检测小目标很重要。

本文的特征金字塔网络（Figure1[d]）把低分辨率、高语义信息的高层特征和高分辨率、低语义信息的低层特征进行自上而下的侧边连接，使得所有尺度下的特征都有丰富的语义信息。这种结构是在CNN网络中完成的，和前文提到的基于图片的金字塔结构不同，而且完全可以替代它。

图像金字塔相关内容点此处

FPN

论文的目标是利用CNN的金字塔层次结构特性（具有从低到高级的语义），构建具有高级语义的特征金字塔。得到的特征金字塔网络（FPN）是通用的，但在论文中，作者先在RPN网络和Fast R-CNN中使用这一成果，也将其用在instance segmentation proposals 中（例如和attention结合的语义分割：Pyramid Attention Network for Semantic Segmentation）。

该方法将任意一张图片作为输入，以全卷积的方式在多个层级输出成比例大小的特征图，这是独立于CNN骨干架构（本文为ResNets）的。具体结构如图Figure 2。

自下而上的路径

CNN的前馈计算就是自下而上的路径，特征图经过卷积核计算，通常是越变越小的，也有一些特征层的输出和原来大小一样，称为“相同网络阶段”。对于原文的特征金字塔，作者为每个阶段定义一个金字塔级别， 然后选择每个阶段的最后一层的输出作为特征图的参考集。 这种选择是很自然的，因为每个阶段的最深层应该具有最强的特征。

自上而下的路径和横向连接

结合低层高分辨率的特征的方法就是，把更抽象，语义更强的高层特征图进行上取样，然后把该特征横向连接（lateral connections ）至前一层特征，因此高层特征得到加强。横向连接的两层特征在空间尺寸上要相同。这样做应该主要是为了利用底层的定位细节信息。

结语：

本文仅对原文中FPN思想进行简单介绍梳理，具体实际应用需详细阅读原文并查阅相关网络结构，例如想明白FPN如何应用在RPN和Fast R-CNN（合起来就是Faster R-CNN），首先要明白Faster R-CNN+Resnet-101的结构。具体细节可以参考：此博客。
